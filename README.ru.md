# RAG Integrity Check

*[English version](README.md)*

Проверяет, что ответы вашего RAG-пайплайна (retrieval-augmented generation) действительно опираются на найденные документы — а не придуманы моделью, не содержат неверных ссылок и не скрывают слепую зону в базе знаний.

## Зачем

RAG должен делать LLM честнее, подкрепляя ответы реальными документами. На практике это работает не всегда: модели всё равно домысливают поверх найденного контекста, ссылаются на источники, которые не говорят того, что им приписывают, и уверенно отвечают на вопросы, по которым в базе знаний нет реального покрытия. Команды обычно узнают об этом от разозлённого пользователя, а не из своих evals.

## Основные идеи

1. **Grounding Checker** — оценивает, насколько ответ реально подкреплён найденными фрагментами, и отмечает предложения, которые выглядят придуманными.
2. **Citation Verifier** — проверяет, что каждая ссылка на источник в ответе действительно есть в документе, на который она указывает.
3. **Chunking Analyzer** — проверяет стратегию разбиения на фрагменты на типовые ошибки: разрезанные посередине предложения, потерянные заголовки, слишком маленькие или большие фрагменты.
4. **Retrieval Auditor** — прогоняет пакет вопросов через ваш retrieval-шаг и находит "слепые зоны": темы без хороших совпадений в документах, где модель, скорее всего, начнёт галлюцинировать.
5. **Reliability Scorer** — объединяет сигналы grounding, цитирования и чанкинга в один объяснимый показатель 0-100 на ответ.
6. **Regression Runner** — прогоняет YAML-набор пар вопрос/ожидаемый источник против реальных ответов пайплайна и отслеживает pass rate во времени, с историей в SQLite.
7. **Reporters** — вывод в text / JSON / JUnit XML, поэтому легко встраивается в любой CI.

## Быстрый старт

```bash
pip install -r requirements.txt

# Оценить один ответ по найденным фрагментам
python -m rag_integrity.cli check --answer answer.json --docs examples/sample_documents.json

# Проверить покрытие retrieval по набору вопросов
python -m rag_integrity.cli audit-retrieval examples/sample_documents.json eval_questions.json

# Прогнать регрессионный набор против реальных ответов пайплайна
python -m rag_integrity.cli regression examples/example_regression_set.yaml responses.jsonl
```

## Файлы

- `rag_integrity/grounding.py` — оценка обоснованности / галлюцинаций
- `rag_integrity/citation_verifier.py` — проверка цитат против документов
- `rag_integrity/chunking_analyzer.py` — анализ границ фрагментов
- `rag_integrity/retrieval_audit.py` — поиск слепых зон базы знаний
- `rag_integrity/scorer.py` — объединённый показатель надёжности 0-100
- `rag_integrity/regression.py` — прогон регрессионного набора из YAML
- `rag_integrity/storage.py` — история прогонов на SQLite
- `rag_integrity/reporters.py` — рендер в text / JSON / JUnit XML
- `rag_integrity/cli.py` — точка входа командной строки
- `tests/` — юнит-тесты для каждого модуля
- `examples/` — пример набора документов и регрессионного набора
