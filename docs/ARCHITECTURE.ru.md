# Архитектура

*[English version](ARCHITECTURE.md)*

## Поток данных

```
ответ + найденные фрагменты --> grounding.check_grounding() --> GroundingResult
ответ + цитаты + документы --> citation_verifier.verify_citations() --> CitationResult
документ + фрагменты --> chunking_analyzer.analyze_chunks() --> ChunkIssue[]
вопросы + результаты retrieval --> retrieval_audit.audit_retrieval() --> BlindSpot[]
(GroundingResult, CitationResult, ChunkIssue[]) --> scorer.score_answer() --> ReliabilityScore
регрессионный набор (yaml) + ответы --> regression.run() --> RegressionReport --> storage.record_regression()
```

`cli.py` объединяет всё это в самостоятельные команды (`check`, `audit-retrieval`, `regression`), каждую из которых можно использовать отдельно или встроить в более крупный eval-пайплайн.

## Модули

| Модуль | Ответственность |
|---|---|
| `grounding.py` | Оценка обоснованности ответа найденными фрагментами; флаги неподтверждённых предложений |
| `citation_verifier.py` | Подтверждение, что цитируемые источники действительно содержат заявленное |
| `chunking_analyzer.py` | Обнаружение проблем границ фрагментов (разрезанные предложения, потерянные заголовки, плохие размеры) |
| `retrieval_audit.py` | Поиск слепых зон базы знаний по набору вопросов |
| `scorer.py` | Объединение всех сигналов в один объяснимый показатель надёжности 0-100 |
| `regression.py` | Прогон регрессионного набора из YAML против ответов пайплайна |
| `storage.py` | Сохранение истории прогонов на SQLite |
| `reporters.py` | Рендер результатов в text / JSON / JUnit XML |
| `config.py` | Централизованная конфигурация через переменные окружения |
| `cli.py` | Команды командной строки, объединяющие всё вместе |

## Принципы проектирования

1. **Никаких скрытых вызовов моделей.** Каждая оценка выводится из явных лексических/статистических проверок против уже имеющихся у вас документов и ответа — полностью офлайн и воспроизводимо.
2. **Объяснимость важнее хитрости.** Каждая оценка сопровождается разбивкой, какие именно проверки прошли или провалились и почему.
3. **История хранится в одном месте (SQLite)**, поэтому отслеживание регрессий и отчётность читают из одного источника истины.
4. **CI прежде всего.** Каждая команда имеет осмысленный exit-код, поэтому работает с любой CI-системой без дополнительной обвязки.
